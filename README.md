# MilePost — Marathon Training Tracker

A native Android app for serious marathon runners. Track your 35-week training plan, sync your runs from Strava, get weekly AI-powered coaching insights, and glance at today's workout without opening your phone.

---

## Features

### Home Screen Widget (Material You)
- **2×2 compact** — today's run type, distance, and race countdown
- **4×2 detailed** — full workout details, last Strava activity, weekly km progress, and days-to-race counter
- Dynamic color: automatically picks up your wallpaper palette on Android 12+
- Phase accent bar changes color as you move through training blocks
- Tap to deep link directly into the app

### Training Plan
- 35-week plan from June 1 2026 to race day (January 17 2027)
- Three target races: Half Marathon (Oct 18), 25K (Dec 20), Full Marathon (Jan 17)
- 8 training phases: Base Building → Aerobic Development → Tempo Introduction → Race Prep → Peak Training → Taper → Race Week → Recovery
- Run types: Easy, Long, Tempo, Intervals, Marathon Pace, Recovery Run, Strides, Race, Gym

### Strava Integration
- OAuth 2.0 via Chrome Custom Tab (never a WebView)
- Auto-sync every 30 minutes via WorkManager
- Cross-references your actual runs against the plan (same date, run type, ±3km tolerance)
- Tap last activity in the widget to open it directly in Strava

### AI Coaching (Claude)
- Weekly coaching report generated by Claude Sonnet
- Race readiness score (0–100)
- Phase-appropriate feedback on volume, consistency, and pacing
- Rate-limited to once per day to avoid unnecessary API calls

### Daily Notifications
- 6:50 AM: today's planned workout
- Pre-race alert when within 7 days of a race
- 9:00 PM: gentle nudge if a run was planned but not logged

## Getting Started

### Prerequisites

- Android Studio Hedgehog or later
- Android SDK 35
- A [Strava API application](https://www.strava.com/settings/api) (free)
- An [Anthropic API key](https://console.anthropic.com) for AI coaching (optional)

### 1. Clone the repo

```bash
git clone https://github.com/aashish-sankar/run-tracker-widget.git
cd run-tracker-widget
git checkout dev
```

### 2. Configure secrets

Create `local.properties` in the project root (already gitignored):

```properties
# Required for Strava sync
STRAVA_CLIENT_ID=your_client_id_here
STRAVA_CLIENT_SECRET=your_client_secret_here

# Optional — enables the AI coaching tab
CLAUDE_API_KEY=sk-ant-...
```

In your Strava API app settings, add this callback URL:

```
marathon://strava/callback
```

### 3. Build and run

```bash
./gradlew assembleDebug
```

Or open the project in Android Studio and run on a device or emulator (API 26+).

## Architecture

```
app/
├── data/
│   ├── local/          # Room entities, DAOs, AppDatabase
│   ├── remote/         # Retrofit APIs, DTOs (Strava + Claude)
│   ├── repository/     # Repository implementations
│   └── workout/        # WorkoutData.kt — full 35-week plan
├── di/                 # Hilt modules (App, Network, Database, Repository)
├── domain/
│   ├── model/          # Pure domain models, no Android deps
│   ├── repository/     # Repository interfaces
│   └── usecase/        # 6 use cases
├── presentation/
│   ├── theme/          # MarathonTheme, Material You colors
│   └── <screen>/       # ViewModel + Screen per feature (6 screens)
├── sync/               # WorkManager workers + notification infra
├── widget/             # GlanceAppWidget, receiver, state manager
└── auth/               # TokenManager, StravaAuthManager
```

**Pattern:** MVVM + Clean Architecture. ViewModels expose `StateFlow<UiState>` sealed classes. The widget reads from a shared DataStore key updated by `WidgetUpdateWorker` every 30 minutes.

## Running Tests

**Unit tests** (JVM, no device needed):

```bash
./gradlew test
```

Covers: `PaceFormatter`, `DateUtils`, `WorkoutData`, token refresh boundary logic, repository contracts, and coaching rate limiting.

**Instrumented tests** (requires emulator or device):

```bash
./gradlew connectedAndroidTest
```

Covers: Room DAOs (in-memory), Today and Dashboard screens with fake repositories via `@TestInstallIn`, and both widget sizes using `runGlanceAppWidgetUnitTest`.

| Weeks | Phase | Focus |
|-------|-------|-------|
| 1–6 | Base Building | Aerobic base, easy effort, introduce gym |
| 7–12 | Aerobic Development | Volume ramp, progressive long runs |
| 13–18 | Tempo Introduction | Lactate threshold work, first tempo blocks |
| 19–20 | Taper / Half Marathon | Reduce volume, race on Oct 18 |
| 21–24 | Race Prep | Rebuild after HM, 25K-specific work |
| 25–30 | Peak Training | Highest volume, marathon pace runs |
